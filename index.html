CSS框模型

概述

CSS框模型(Box Model)规定了元素框处理元素内容、内边距、边框和外边距的方式.

框模型图示:

元素的最内部分是实际的内容, 直接包围内容的是内边距. 内边距呈现了元素的背景. 内边距的边缘是边框. 内边距以外是外边距，外边距默认是透明的，因此不会遮挡其他的任何元素.

注意:定义的背景声明将被应用于边框以内的区域，即内边距和内容组成的区域.

内边距、边框和外边距都是可选的，默认值是零.

在CSS中，width和height指的是内容区域的宽度和高度. 增加内边距、边框和外边距不会影响内容区域的尺寸，但是会增加元素框的总尺寸.

注意:内边距、边框和外边距可以应用于一个元素的所有边，也可以应用于单独的边.

注意:外边距可以是负值，而且很多情况下都要使用负值的外边距.

CSS内边距

  • 元素的内边距在边框和内容区之间。控制该区最简单属性就是padding属性.

  • CSS padding属性定义元素边框与元素内容之间的空白区域.

  • CSS padding属性定义元素的内边距.padding 属性接受长度值或百分比，但不允许使用负值.

  • 使用四个单独的属性，分别设置上、右、下、左内边距:

内边距的百分比数值:

可以为元素的内边距设置百分数值.百分数值是相对于其父元素的width计算的, 这一点与外边距一样.所以，如果父元素的width改变，他们也会改变.

注意:上下内边距与左右内边距一致; 即上下内边距的百分数会相对于父元素宽度，而不是相对于高度.

CSS边框

  • 元素的边框(boder)是围绕元素内容和内边距的一条或多条线.

  • CSS border属性允许你规定元素边框的样式、宽度和颜色.

边框与背景

CSS规范指出，边框绘制在”元素的背景上”.这很重要, 因为有些边框是”间断的”(例如点线边框和虚线框),元素的背景应该出现在边框的可见部分之间.

定义单边样式

为某一个边设置边框样式,而不是设置所有4个边的边框样式,可以使用下面的单边框样式属性:

  • border-top-style
  • border-right-style
  • border-bottom-style
  • border-left-style

边框的宽度

通过border-width属性为边框指定宽度.

为边框指定宽度有两种方法:

  • 直接使用长度单位指定: 2px或0.1em;

  • 使用关键字 thin、medium(默认值)和thick.

没有边框

默认情况下border-style是none，也就是没有边框，如下的定义，将不会显示边框:

如果border-style设置为none，即边框根本不存在，那么边框就不可能有宽度，因此边框宽度自动设置为0，而不论您原先定义的是什么。

由于 border-style 的默认值是 none，如果没有声明样式，就相当于 border-style: none. 因此，如果您希望边框出现，就必须声明一个边框样式.

边框的颜色

设置边框颜色非常简单. CSS使用一个简单的border-color属性, 它一次可以接受最多四个颜色值.

透明边框

有些时候希望创建一个不可见的边框.

这种透明边框相当于内边距，因为元素的背景会延伸到边框区域（如果有可见背景的话）.

CSS外边距

设置外边距的最简单的方法就是使用margin属性

margin属性接受任何长度单位, 可以是像素、英寸、毫米或em.

外边距的合并问题

外边距合并指的是，当两个垂直外边距相遇时，它们将形成一个外边距. 合并后的外边距的高度等于两个发生合并的外边距的高度中的较大者.

注意:只有普通文档流中块框的垂直外边距才会发生外边距合并. 行内框、浮动框或绝对定位之间的外边距不会合并.

CSS语法学习 一

CSS的由来

  • 互联网是用来传播和分享信息的.
  • 在互联网发展的早期,只有文本信息在互联网上传播，类似于飞鸽传输信件，而这个特殊的信件就是HTML文件.
  • 为了信件更加具有可读性, HTML文件引入了丰富的标签. 信件传到浏览器终端以后，由浏览器负责拆封并展示信息.
  • 问题是互联网能够传播的信息越来越丰富, 从文字到图片到视频……同时,人们对信件的美观要求也越来越高.为了方便开发, 人们需要将信件的内容和样式(表现)分离开来,也就是Cascading Style Sheets.

css(层叠样式表),指的是多个样式定一可以层叠为一.

多重样式的重叠顺序

当同一个HTML元素被不止一个样式定义时,会使用哪个样式呢

一般而言，所有的样式会根据下面的规则层叠于一个新的虚拟样式表中，其中数字4拥有最高的优先权

  1. 浏览器缺省设置

  2. 外部样式表\\\

  3. 内部样式表

  4. 内联样式(内容与表现混杂在了一起，谨慎使用!)

其中样式表的优先级:

类选择器 < 类派生选择器 < ID选择器 < ID派生选择器

派生选择器也叫上下文选择器，就是通过元素在其位置的上下文关系来定义样式，以达到更加简洁的样式定义目的. 简单点说，在选择器语法中要出现位置关系。常见的派生选择器包含三种:

  • 后代选择器,选择包含在A元素中的所有B元素, 不关心具体层级(空格连接 h1 p em {color:red;})
  • 子元素选择器,选择包含在A元素中的直接子元素B(大于号连接 html > body {color:red;})
  • 兄弟选择器, 选择A元素的第一个最相邻B元素，并且AB元素必须要有同一个兄弟元素(加号 h1 + p{color:red})
